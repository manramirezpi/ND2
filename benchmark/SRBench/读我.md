SRBench 的精简版本，移除了不需要的测试脚本和 Github 自动 workflow 的相关脚本

# 配置
- 需要将 pmlb 数据集下载到 benchmark/SRBench/pmlb 路径下，使得 pmlb/{name}/{name}.tsv.gz 是对应的数据，pmlb/{name}/metadata.yaml 是对应的说明
- 需要配置环境变量 `PYTHONPATH=.`，可以通过执行 `export PYTHONPATH=.` 临时设置，或者在 .vscode/settings.json 中设置
```json
{
    "terminal.integrated.env.linux": { "PYTHONPATH": "." },
    "terminal.integrated.env.osx": { "PYTHONPATH": "." },
    "terminal.integrated.env.windows": { "PYTHONPATH": "." },
}
```

# 运行
通过 `python benchmark/SRBench/analyze.py {dataset} -ml {algorithm} -n_trials {n_trials} -njobs {njobs} --local --sym_data -script benchmark/SRBench/evaluate_model -results result/srbench` 执行测试脚本，根据指定的（多个）datasets，（多个）algorithms，以及（多个）seeds 生成一系列测试指令，移除已有的结果后通过多进程并行地调用 benchmark/SRBench/evaluate_model.py，并将运行结果保存到 `result/srbench/{dataname}/{dataname}_{algorithm}_{seed}.json`
- `{dataset}`: 可以是 `./benchmark/SRBench/pmlb/feynman_II_8_31` 这样的单个 dataset，也可以用 `./benchmark/SRBench/pmlb/feynman_*` 批量匹配所有 feynman 数据集
- `-ml {algorithm}`: 需要对应于 `./methods/{algorithm}/regressor.py` 中的命名，regressor.py 中需要定义一个 `est`，通过 `.fit(X,y)` 方法拟合给定的数据
- `-n_trials {n_trials}`: 指定每个实验的重复次数，每次重复使用 seeds.py 中定义的相同的种子
- `-njobs {njobs}`: 指定并行执行的进程数量
- `--local`: 表明在本地而非集群上运行
- `--sym_data`: 在 feynman 或 strogatz 数据集上测试时必须指定这一标记

还可以指定最长运行时间 & 是否覆盖已有结果，详见 analyze.py
